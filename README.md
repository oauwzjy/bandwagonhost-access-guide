## 三步解决 DNS 污染问题

如果换了镜像域名还是进不去，问题可能出在本地 DNS 缓存或 DNS 服务器上。按顺序试：

**第一步：刷新本地 DNS 缓存**

Windows 系统：按 Win+R 打开运行，输入 `cmd`，然后执行：


ipconfig /flushdns


macOS 系统：打开终端，执行：


sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder


刷完之后重新在浏览器里输入 bwh81.net，多数情况这一步就解决了。

**第二步：临时换成公共 DNS**

如果刷缓存不够用，把网络设置里的 DNS 服务器改成 Google 的 8.8.8.8 或 Cloudflare 的 1.1.1.1。这两个公共 DNS 一般不会有污染问题。

Windows：打开"网络和共享中心" → 更改适配器设置 → 右键你的网络连接 → 属性 → IPv4 → 手动填写 DNS。

**第三步：修改 hosts 文件（技术用户备选方案）**

有一定基础的用户可以直接在系统 hosts 文件里写入搬瓦工镜像域名和 IP 的对应关系，绕过 DNS 解析。搬瓦工官方文章里也提供了对应的 IP 地址，填进去之后不管 DNS 怎么被污染都影响不了本地的解析结果。
